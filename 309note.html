CSS BASICS

  • for appearance, layout, presentation
  • can be written in HTML directly, but not recommended, use separate CSS file and embedded in the head of an HTML page
  • work with rules and selectors (selector { property: value;})
  • a page can link to multiple CSS files, in case of conflict: latter > f ormer; detail > general
  • comment: /*...*/
  • use class and id in html, make CSS easier to target
  • states and pseudo class (a:link { color: #FF0000; } /* unvisited link */)
  • check use W3C
CSS types and properties
  • color, three ways to define
    1. conventional name:aqua, black, blue, fuchsia, gray, green, lime, maroon, navy, olive, purple, red, silver, teal, white
    2. RGB code: (0,0,0)~(255,255,255)
    3. hex code: #000000~#FFFFFF
  • font size, three ways to define
    1. Unit: pixels (16px) vs. point (16pt) vs. m-size (1.16em) vs. root em (16rem)
    2. Vague font sizes: xx-small~;arger
    3. Percentage: 90%, 120%
  • a lot about font (font-family, style, weight, align...)

    we can apply multiple fonts to the same text, they are priority ordered (the latter fo nt would be used only if the former font is not found)

  • a lot about background (background-color, iamge, repeat...)

GitHub

  • distributed version Cntrol, waht does that mean?
  • four stages: untracked, unmodified, modified, staged
  • fork/ remote repository

    When you fork a repository, you make your own copy of other's repository to your own GitHub account. Any repos itory that is not on your local computer is a remote repository

  • clone

    When you clone a repository, you get a copy of repository from github to your local device.

  • branches

    When you create a branch, you copy the repository in order to change some contents, by this way you do not affect the master repository directly when doing any changes.

  • pull/push

    Basically, pull and push request works just like commit and update, but it is used when you want to finally change your remote repository since you have done on your local working copy

HTML5/CSS3

HTML5, some rules and new features

  • semantic HTML: all tags should be meaningful, rather than just for styling.

    e.g.: use <em> for bold because you want to emphasize rather than <b> which b stands for bold, there are a lot of new tags rather than div

  • audio and video control

    New feature in HTML5, look at this link to see how it works. It includes two sources (mp4, ogg) and an alt text, in case the web browser does not support the video

  • canvas

    New feature in HTML5, use to draw grphic by some script language such as JavaScript

  • drag and drop
  • web storage

    website stores some data on client's computer, cookies, easy to use

  • indexedDB

    also used to store data, it has complicate asynchronous API and is better to deal with complex data

  • server-sent events

    not every browser supports, just use JavaScript

  • geological API
